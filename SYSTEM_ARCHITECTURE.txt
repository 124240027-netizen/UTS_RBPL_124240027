🔐 AUTHENTICATION FLOW:

1. REGISTRATION:
   register.php → Validate input → Hash password (bcrypt) → Insert to DB

2. LOGIN:
   login.php → Check email → Verify password → Create session → Redirect

3. SESSION MANAGEMENT:
   $_SESSION['user_id']   → User ID
   $_SESSION['user_name'] → User name
   $_SESSION['user_role'] → admin/customer

4. ACCESS CONTROL:
   is_logged_in()  → Check if user is logged in
   is_admin()      → Check if user is admin
   redirect()      → Redirect if not authorized

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 BOOKING WORKFLOW:

1. CUSTOMER SELECTS PACKAGE
   └─ booking.php (Package selection & form)

2. FILL BOOKING DETAILS
   └─ Event date, type, location, notes

3. VALIDATE & CALCULATE PRICE
   └─ Check date validation (must be future)
   └─ Get package price

4. CREATE BOOKING RECORD
   └─ INSERT into bookings table
   └─ Status: pending

5. REDIRECT TO PAYMENT
   └─ payment.php (payment_id provided)

6. PROCESS PAYMENT
   └─ Select payment method
   └─ Confirm transaction
   └─ UPDATE booking status: payment_status = 'paid'

7. DISPLAY SUCCESS
   └─ Show booking confirmation
   └─ Provide order number

8. TRACK IN DASHBOARD
   └─ dashboard.php (View all bookings)
   └─ booking-detail.php (View single booking)

9. ADMIN APPROVAL
   └─ admin/bookings.php
   └─ Update status: pending → confirmed/rejected

10. COMPLETION & REVIEW
    └─ After event: admin updates status to 'completed'
    └─ Customer can give rating & comment
